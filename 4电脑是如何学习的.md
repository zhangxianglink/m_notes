# 电脑是如何学习的：从 Loss 到 Backpropagation

*(面向软件开发工程师的 AI 科普)*

> **📝 导读：** 上一章我们讲了神经元的基本公式：`输出 = 激活函数(输入 * 权重 + 偏置)`。\
> 但新的问题马上就来了：如果权重和偏置一开始都是随机数，模型为什么最后能学会识别人脸、理解文字、甚至写代码？
>
> 答案是：它不是“突然开悟”，而是在一次次试错中，靠 **损失函数、梯度、学习率、反向传播** 慢慢把几亿到几千亿个参数调到更合适的位置。

***

## 一、 先承认自己算错了：损失函数（Loss Function）

> **💡 原文回放**：因为权重是随机的所以结果是对的吗？这时候就要引入一个损失函数。

这是一个非常好的切入点。

模型刚初始化时，权重和偏置通常是随机值，所以它的预测大概率是不准的。\
既然不准，我们就得先回答一个问题：

**“它到底错了多少？”**

这就是 **损失函数（Loss Function）** 的作用。\
它不是拿来“训练”的，而是拿来**量化当前预测和正确答案之间的差距**。

最简单的损失函数之一就是平方误差：

```text
Loss = (预测值 - 目标值)^2
```

平方有两个好处：

- 把负数误差变成正数，避免“正负抵消”
- 让大错误受到更重的惩罚

### 一个更正后的例子

> **⚠️ 勘误提示**：原稿里的这组计算有误。
>
> - `20 * 0.5 + 1` 不是 `10`，而是 `11`
> - 所以后面的损失值和梯度值都要一起修正

假设现在有一个最简单的线性神经元：

- 输入 `x = 20`
- 权重 `w = 0.5`
- 偏置 `b = 1`

为了先看清训练原理，我们这里**先不加激活函数**，直接看线性输出：

```text
预测值 y = w * x + b
       = 0.5 * 20 + 1
       = 11
```

如果目标值是 `15`，那么损失函数就是：

```text
Loss = (11 - 15)^2 = 16
```

这一步可以理解为：

- 神经网络先给出答案：`11`
- 标准答案是：`15`
- 系统打分后告诉它：`你这次错了 16 分`

## 二、 梯度不是“答案”，而是“修改建议”

> **💡 原文回放**：当知道了差异之后，就可以根据差异来调整权重和偏置。这里面就引入了一个概念，梯度。

知道“错了多少”还不够，我们还得知道：

**“应该往哪边改？”**

这时候就轮到 **梯度（Gradient）** 出场了。

### 1. 什么是梯度？

如果只有一个参数，梯度可以近似理解为“当前点的斜率”。\
如果有很多参数，梯度其实是一个**由很多偏导数组成的向量**，告诉我们：

- 哪个参数改动最敏感
- 往哪个方向调，会让损失下降得更快

对工程师来说，梯度可以类比成：

**编译器或测试框架给你的报错定位信息。**

它不直接帮你把 Bug 修好，但它会告诉你：

- 问题主要出在哪
- 朝哪个方向改更可能修好

### 2. 为什么要用“偏导数”？

因为神经网络里不是只有一个参数，而是有很多个。

比如上面的线性模型里，至少就有两个参数：

- 权重 `w`
- 偏置 `b`

所以我们要分别问：

- 如果我只改 `w`，Loss 会怎么变？
- 如果我只改 `b`，Loss 会怎么变？

这就是：

```text
∂Loss/∂w
∂Loss/∂b
```

它们叫做 **偏导数**。

***

## 三、 手算一次梯度：为什么原稿里的方向写反了？

> **💡 原文回放**：`(预测值-目标值) * 输入 * 2` 是对权重的偏导数。
> **⚠️ 勘误提示**：这句话只对“权重梯度”成立，不适用于偏置；而且原稿后面对符号方向的解释也反了。

我们继续用刚才的例子：

```text
y = w * x + b
Loss = (y - target)^2
```

已知：

- `x = 20`
- `w = 0.5`
- `b = 1`
- `y = 11`
- `target = 15`

### 1. 对权重求偏导

```text
∂Loss/∂w = 2 * (y - target) * x
         = 2 * (11 - 15) * 20
         = -160
```

### 2. 对偏置求偏导

```text
∂Loss/∂b = 2 * (y - target)
         = 2 * (11 - 15)
         = -8
```

注意这里非常关键的一点：

> **梯度是负数，不代表参数应该减小；恰恰相反，在梯度下降里，参数更新公式是** **`参数 = 参数 - 学习率 * 梯度`。**

所以如果梯度是负数，减去一个负数，参数反而会**增加**。

这也是为什么原稿里“梯度是 -100，说明权重应该减少”这个判断是不对的。

## 四、 学习率：每次改多少？

> **💡 原文回放**：有了方向，接下来就是调整的力度了，叫做学习率。

方向知道了，还差最后一个问题：

**“这次改多少比较合适？”**

这就是 **学习率（Learning Rate）**。

更新公式通常写成：

```text
新参数 = 旧参数 - 学习率 * 梯度
```

假设学习率：

```text
lr = 0.001
```

那么：

```text
w_new = 0.5 - 0.001 * (-160) = 0.66
b_new = 1   - 0.001 * (-8)   = 1.008
```

更新后的预测值：

```text
y_new = 0.66 * 20 + 1.008 = 14.208
```

新的损失：

```text
Loss_new = (14.208 - 15)^2
         ≈ 0.627
```

可以看到，Loss 从 `16` 迅速下降到了 `0.627`。\
这就说明：**这一步更新是有效的。**

### 学习率为什么不能太大，也不能太小？

- **太大**：像下山时一步跨太远，可能直接跨过最低点，来回震荡，甚至发散
- **太小**：像蚂蚁挪步子，虽然方向对，但训练会非常慢

```text
算一下错了多少   -> Loss
算一下该往哪调   -> Gradient
调一点点         -> Gradient Descent
重复很多次       -> Training
```

## 五、 梯度下降：本质是在“地形图”上找最低点

> **💡 原文回放**：梯度下降是我们的目标，一条曲线，我们的目标就是使这条曲线的斜率最小。
> **⚠️ 勘误提示**：更准确地说，目标不是“让斜率最小”，而是**让损失函数尽量小**；在最低点附近，梯度会接近 0。

可以把 Loss 想象成一张地形图：

- 山顶表示误差很大
- 山谷表示误差很小

模型训练的过程，就像一个球在山坡上往下滚：

```text
        山顶
         /\
        /  \
       /    \__
      /        \___  <- 更低的损失
```

梯度告诉我们当前坡最陡的方向。\
梯度下降做的事情就是：

**朝着“让 Loss 变小”的方向走一小步。**

如果参数只有一个，这是一条曲线。\
如果参数有几亿个，这是一个你根本画不出来的超高维曲面。

## 六、 前向传播与反向传播：一层层算，一层层改

> **💡 原文回放**：前向传播是从输入层到输出层，反向传播是从输出层到输入层。

这个表述方向上是对的，但可以再讲得更清楚一点。

### 1. 前向传播（Forward Propagation）

前向传播就是：

- 输入数据进来
- 每一层都做自己的乘法、加法、激活函数
- 最后得到一个预测结果
- 再和标准答案比较，算出 Loss

可以把它理解成：

**“先跑一次程序，看看输出错了多少。”**

### 2. 反向传播（Backpropagation）

反向传播不是简单地“把梯度往前传”，而是利用 **链式法则（Chain Rule）**，把最终误差对每一层参数的影响一层层拆开算出来。

也就是说：

- 输出层先知道自己错了多少
- 然后把这份“责任”往前一层层分摊
- 每一层都算出自己那部分参数应该怎么调

这很像一次大型事故复盘：

- 最终线上事故发生了
- 先看最终接口输出
- 再沿着调用链一层层往回查
- 定位到底是哪一层、哪个参数贡献了更多错误

### 3. 为什么反向传播一定要用链式法则？

这是因为神经网络本质上不是一个单独的公式，而是**很多层函数套很多层函数**。

比如一个极简的两层网络，可以写成：

```text
z1 = w1 * x + b1
a1 = ReLU(z1)
z2 = w2 * a1 + b2
y  = z2
Loss = (y - target)^2
```

如果你把这些式子连起来看，会发现：

```text
Loss
  依赖 y
  y 依赖 z2
  z2 依赖 a1
  a1 依赖 z1
  z1 依赖 w1
```

现在问题来了：\
我们真正想知道的是，**最前面那层的参数** **`w1`** **到底该怎么改？**

但 `w1` 并不会直接出现在 `Loss` 的最终公式表面上，它是通过很多中间变量，间接影响了最终损失。\
所以你不能一句话直接“看出来” `w1` 对 `Loss` 的影响，只能顺着这条依赖链一段一段往回算。

这就是链式法则：

```text
∂Loss/∂w1
= ∂Loss/∂y
× ∂y/∂z2
× ∂z2/∂a1
× ∂a1/∂z1
× ∂z1/∂w1
```

它的直觉含义是：

- `w1` 改一点点，会让 `z1` 变多少
- `z1` 变一点点，会让 `a1` 变多少
- `a1` 再传到下一层，会让 `z2` 变多少
- `z2` 最终会让 Loss 变多少

把这些“局部影响”乘起来，才能得到 **“最早这个参数，最终对损失造成了多大影响”**。

### 4. 不用链式法则会怎么样？

如果不用链式法则，你就会遇到两个大问题：

- **第一，根本算不出深层参数的梯度。** 因为深层网络是函数嵌套，不拆开就没法知道前面某个参数是怎么影响最后输出的。
- **第二，计算会极其低效。** 链式法则允许我们复用后一层已经算出来的梯度，像动态规划一样逐层复用结果，这正是反向传播高效的关键。

换成工程师更熟悉的说法：

- 前向传播像是一次正常请求调用链：`Controller -> Service -> DAO -> DB`
- 反向传播像是线上事故发生后，顺着调用链逆向排查影响
- 链式法则就是这套排查过程里的**依赖传递规则**

没有链式法则，你只能暴力重算每个参数对最终错误的影响；\
有了链式法则，你就能把“最终出了多大错”这件事，一层层拆账，精确算到每个参数头上。

### 5. 训练的一次完整循环

```text
输入数据
  -> 前向传播
  -> 得到预测值
  -> 计算损失 Loss
  -> 反向传播求梯度
  -> 更新参数
  -> 下一轮
```

这就是训练最基本的闭环。

## 七、 Epoch、Batch、GPU：训练不是“顿悟”，是体力活

> **💡 原文回放**：没有顿悟，有的只是一轮一轮的训练，直到模型的误差最小化。

这句话说得很好，我保留下来并展开一下。

### 1. Epoch 是什么？

一个 **Epoch** 指的是：

**整个训练集被模型完整看过一遍。**

比如你有 100 万条训练数据，模型把这 100 万条都学完一次，这就叫 1 个 epoch。

### 2. Batch 是什么？

现实里不可能每次只拿 1 条数据训练，也不可能一次把全部数据塞进显存。\
所以通常会把数据切成一个个小批次，叫 **Batch**。

比如：

- 总数据量：100 万条
- 每个 batch：100 条

那么一次 epoch 就要训练 10000 个 batch。

### 3. GPU 为什么重要？

因为神经网络底层主要是在做：

- 矩阵乘法
- 向量加法
- 并行计算

而 GPU 最擅长的就是大规模并行浮点运算。\
它不是“更聪明”，只是**更适合做这种密集计算**。

## 八、 数据为什么这么重要？以及哪些原稿说法需要修正

> **💡 原文回放**：当前人类历史上所有的文本数据都已经训练过了。数据量多，超过神经网络参数的数量，模型就开始学习规律，而不是背诵数据。
> **⚠️ 勘误提示**：这两句都不够严谨。

### 1. 不是“全人类所有文本都训练过了”

实事求是地说，我们通常只能确认：

- 大模型会使用**公开可得、授权可用、人工标注、合成生成**等多种来源的数据
- 数据规模非常庞大
- 数据清洗、去重、过滤和标注都很重要

但“全人类历史上所有文本都训练过了”这种说法是不能严谨成立的。

### 2. 为什么数据量大很重要？

因为如果数据太少，模型就容易：

- 记住训练样本
- 学不会泛化规律
- 在训练集表现很好，在新数据上表现很差

这就是 **过拟合（Overfitting）**。

这很像考试：

- 只背 10 道原题，碰到变形题就不会
- 但如果你刷了大量不同题型，并真正理解规律，泛化能力就强很多

### 3. “数据量超过参数量”不是铁律

原稿里这句启发方向是对的，但表达太绝对。

真实情况更复杂，模型效果通常取决于很多因素共同作用：

- 数据量
- 数据质量
- 数据分布是否合理
- 模型结构
- 优化算法
- 正则化方法
- 训练策略

不是简单一句“数据比参数多，就一定能学会规律”。

### 4. 人工标注到底有什么用？

这一部分可以严谨地说：

- 在预训练阶段，模型大量从文本中自监督学习
- 在对齐阶段，人类标注者会参与偏好排序、质量评估、安全对齐等工作

所以“高质量标注很重要”是对的，\
但具体哪个公司是不是由管理层亲自参与标注，如果没有可靠公开来源，就不应该当成事实写进科普文档。

***

## 九、 训练的本质：不是悟道，而是反复修参数

把整件事浓缩成最朴素的话，其实就是这四步：

1. **先算出答案**
2. **再看看错了多少**
3. **再算该往哪边改**
4. **最后只改一点点**

然后不断重复。

```text
预测 -> 计算误差 -> 计算梯度 -> 更新参数 -> 再预测
```

> **💡 总结**：\
> 神经网络的学习，不是“突然理解世界”，而是把海量样本喂给一个初始随机的数学系统，再用损失函数衡量错误、用梯度指出方向、用学习率控制步长、用反向传播把责任分摊到每一层，最后通过一轮又一轮训练，把参数慢慢拧到更合理的位置。\
> 从这个角度看，AI 的学习更像是一个超大规模的自动调参系统，而不是神秘的黑箱顿悟。
