# Attention 机制：Transformer 为什么能“一眼看全局”
*(面向软件开发工程师的 AI 科普)*

> **📝 导读：** 如果说前面几章在讲“神经网络会算”，那这一章就在讲“Transformer 为什么突然变聪明了”。  
> 在 `Attention Is All You Need` 这篇论文出现之前，处理文本主流靠的是 RNN/LSTM，处理图片主流靠的是 CNN。Transformer 的革命性不在于“会做乘法”，而在于它改变了**信息在序列里流动的方式**。

***

## 一、 Transformer 之前：RNN 和 CNN 为什么不够用？

> **💡 原文回放**：在 transformer 架构出现之前，是 RNN 循环神经网络，CNN 卷积神经网络的天下。

这个总方向是对的，但可以更准确一点。

### 1. RNN / LSTM 的优点和痛点

RNN（循环神经网络）擅长处理序列数据，比如：

- 文本
- 语音
- 时间序列

它的工作方式很像一个 `for` 循环：

```python
state = init_state
for token in sequence:
    state = update(state, token)
```

这带来两个问题：

- **不能很好并行**：第 10 个 token 的计算，得等前 9 个算完
- **长距离依赖难**：一句话太长时，前面信息传到后面容易衰减

LSTM 和 GRU 是对 RNN 的改良版，确实缓解了“记不住”的问题，但没有从根上解决：

- 长序列训练慢
- 并行性差
- 远距离依赖仍然难学

### 2. CNN 为什么不是文本建模的终极答案？

CNN 很擅长图像，是因为图片天然是网格结构，局部区域很重要。  
它像一个滑动窗口扫描器，专门提取局部模式。

但如果用 CNN 处理文本，它更像是在看“局部 n-gram 模式”：

- 看相邻几个词的组合还行
- 但要建模很远位置之间的关系，就没那么自然

比如这句话：

```text
小明把书借给了小红，因为她明天要考试。
```

这里“她”指的是谁，需要全局理解。  
RNN 需要慢慢传，CNN 需要堆很多层，而 Transformer 的思路是：

**别绕了，直接让每个词都能看见其他词。**

## 二、 Attention 的直觉：我在看谁？

> **💡 原文回放**：它看文字不是按顺序看，而是按字之间的关系来处理，一瞬间把所有的文字都看完了。

这个直觉非常好，可以再严谨一点：

- Transformer **不是完全不看顺序**
- 它是**不靠递归顺序处理**
- 但会通过**位置编码（Positional Encoding）**补充位置信息

Attention 的核心问题其实只有一句话：

**“我在处理当前这个 token 时，应该重点参考句子里的谁？”**

比如：

```text
The animal didn't cross the street because it was too tired.
```

当模型看到 `it` 时，它需要判断 `it` 更可能指向谁。  
Attention 的作用，就是给上下文里其他 token 打分，看看哪些更值得参考。

可以把它想成一个“动态依赖分析器”：

- 当前 token 像是一个查询请求
- 其他 token 像是可供参考的上下文项
- 模型自动算出“谁对我最重要”

## 三、 Q、K、V 到底是什么？

> **⚠️ 勘误提示**：原稿里有几处容易混淆：
> - `Q/K/V` 不是 token 天生自带的三种向量，而是由输入表示乘上不同权重矩阵后得到的
> - 注意力矩阵不是“行是 K、列是 V”，而是 **Query 和 Key 的匹配分数矩阵**
> - `V` 也不是“初始向量本身”，而是输入表示经过 `Wv` 线性变换后的结果

假设输入序列经过 `token embedding + positional encoding` 之后，得到矩阵：

```text
X ∈ R[n, d_model]
```

其中：

- `n` 是序列长度
- `d_model` 是隐藏维度

接下来模型会用三组可学习的权重矩阵，把 `X` 映射成三种不同的视角：

```text
Q = XWq
K = XWk
V = XWv
```

### 它们的直觉含义

- **Q (Query)**：我当前在找什么信息
- **K (Key)**：我这里有什么信息可供别人匹配
- **V (Value)**：如果你真的关注我，我应该把什么信息传给你

工程师类比：

- `Q` 像查询条件
- `K` 像索引键
- `V` 像真正的数据载荷

也就是说，Attention 不只是“看谁相关”，还要决定：

- 谁值得看
- 看了之后把什么内容拿过来

## 四、 Scaled Dot-Product Attention：公式到底在干嘛？

原论文最经典的公式是：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

我们一段一段拆开看。

### 1. `QK^T`：先算“相关性分数”

`Q` 和 `K` 做点积，本质上是在问：

**“当前 token 的查询方向，和其他 token 的键方向有多对得上？”**

如果对得上，分数就高；对不上，分数就低。

比如一句话里：

```text
苹果公司发布了新手机，它卖得很好。
```

当模型处理 `它` 时，`Q(它)` 会和所有词的 `K` 做匹配。  
如果 `苹果公司` 的 key 和 `它` 的 query 更对齐，那么它们的注意力分数就更高。

### 2. 为什么要除以 `sqrt(d_k)`？

> **⚠️ 勘误提示**：原稿把 `d` 说成“缩放因子”，方向对，但最好讲清楚原因。

当向量维度变大时，点积的数值容易变得很大。  
如果直接把很大的分数丢给 `softmax`，会导致：

- 某几个位置权重几乎变成 1
- 其他位置几乎变成 0
- 梯度变差，训练不稳定

所以要除以 `sqrt(d_k)`，让数值尺度更平稳。

这一步更像是：

**给注意力分数做一个数值稳定化处理。**

### 3. `softmax(...)`：把分数变成“注意力权重”

`softmax` 的作用是：

- 把任意实数分数转成非负权重
- 并且让这些权重加起来等于 1

这样模型就相当于在说：

```text
我对 A 分配 60% 注意力
我对 B 分配 30% 注意力
我对 C 分配 10% 注意力
```

> **⚠️ 勘误提示**：原稿里提到“为了非线性一般会去掉负数”，这个表述不准确。  
> Transformer 里并不是靠“去掉负数”实现注意力非线性，而是通过 `softmax` 把分数归一化为概率分布。

### 4. 最后的 `... V`：把信息加权汇总

有了注意力权重之后，模型会对所有 `V` 做加权求和：

```text
output_i = a1 * V1 + a2 * V2 + ... + an * Vn
```

所以 Attention 的输出不是“注意力矩阵本身”，而是：

**当前 token 从其他 token 那里汇总来的上下文信息。**

这一步特别重要。

Attention 不是只做“谁和谁像”的分析图，它最终目的是：

**把别的 token 的信息，按权重融合进当前 token 的表示里。**

## 五、 直观理解：Attention 到底更新了什么？

原稿里有一句话非常接近本质：

> “把各种影响信息，按照权重更新到初始向量，使它丰富，能代表不同的含义。”

这句话可以保留，但更严谨地说：

- 每个 token 一开始只有一个**初始表示**
- 经过 attention 后，它会吸收其他 token 的相关信息
- 于是这个 token 的表示变成了一个**上下文化表示（contextualized representation）**

比如 `apple`：

- 在词表里，它初始只是一个固定 embedding
- 在句子 `I ate an apple` 里，它会变成“水果方向”的表示
- 在句子 `Apple released a new chip` 里，它会变成“公司方向”的表示

这就是 Attention 真正厉害的地方：

**不是给词一个固定含义，而是根据上下文动态改写它的含义。**

## 六、 Input Embedding 与 Positional Encoding：为什么模型知道“顺序”？

> **💡 原文回放**：首先是文字要转换为 token，token 要转换为嵌入向量 embedding vector。

这一步是对的，但还缺了一个关键部件：**位置**。

### 1. Input Embedding

文本先经过分词器变成 token id：

```text
["我", "爱", "北京"] -> [101, 502, 930]
```

然后通过 embedding table 查表，变成浮点向量：

```text
token_id -> embedding vector
```

这一步解决的是：

**“把离散符号变成连续向量。”**

### 2. Positional Encoding

但光有 embedding 不够，因为：

```text
我 爱 你
你 爱 我
```

如果只有 token embedding，而没有位置信息，这两句话看起来几乎是同一堆 token。

所以 Transformer 要额外告诉模型：

- 谁在第 1 个位置
- 谁在第 2 个位置
- 谁在第 3 个位置

原始论文里使用的是**正弦/余弦位置编码**：

```text
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

实际直觉不用被公式吓到，你只要记住：

**Transformer 必须额外注入位置信息，否则它只知道“有哪些词”，不知道“谁在前谁在后”。**

输入到第一层 Transformer 的向量，通常就是：

```text
X = TokenEmbedding + PositionalEncoding
```

## 七、 Multi-Head Attention：为什么不是只用一个注意力头？

如果只有一个 attention head，模型每次只能用一种“关注方式”看句子。

但真实语言关系很多种：

- 指代关系
- 语法关系
- 修饰关系
- 主谓宾关系
- 长距离依赖

所以 Transformer 的做法是：

**并行开多个注意力头，让模型从不同子空间、不同角度同时看这句话。**

每个 head 都有自己的一套：

- `Wq`
- `Wk`
- `Wv`

于是不同 head 可能分别学会：

- 谁是主语
- 谁修饰谁
- 哪些词语义相近
- 哪些位置需要重点看

工程师类比：

- 一个头像“语法分析器”
- 一个头像“指代消解器”
- 一个头像“主题提取器”

最后再把所有 head 的输出拼接起来，经过一次线性变换融合。

这就是 **Multi-Head Attention** 的意义：

**不是重复算很多遍，而是让模型并行学习多种关系模式。**

## 八、 残差连接、LayerNorm、前馈网络：Transformer 不止是 Attention

> **⚠️ 勘误提示**：原稿里有一个容易误导的点：Transformer 不是“只有 attention”。  
> 原论文标题叫 `Attention Is All You Need`，意思是“不再需要 RNN/CNN 来做序列建模的主干”，不是说整个架构里除了 Attention 什么都没有。

一个典型的 Transformer block，除了 Attention，还包括下面这些关键零件。

### 1. 残差连接（Residual Connection）

形式上很像：

```text
output = x + SubLayer(x)
```

它的作用是：

- 保留原始信息
- 让深层网络更容易训练
- 缓解梯度传递困难

程序员类比：

它像是一个“旁路保底通道”。  
就算子模块学得不好，原始输入也还能直接传过去，不至于把信息完全搞丢。

### 2. Layer Normalization

LayerNorm 会对每个 token 的隐藏表示做归一化，让数值分布更稳定。

作用主要是：

- 稳定训练
- 加快收敛
- 减少不同层之间数值尺度漂移

你可以把它理解成：

**每层处理前后，先把数值范围“整理一下”，别让某些维度突然爆掉。**

### 3. Position-wise Feed-Forward Network（前馈网络，FFN）

Attention 负责“信息交换”，FFN 负责“每个位置自己做更深的非线性加工”。

通常是两层线性层加激活函数：

```text
FFN(x) = W2 * GELU(W1 * x + b1) + b2
```

它对每个 token 位置独立执行，但参数是共享的。

直觉上：

- Attention：让我看看别人说了什么
- FFN：我结合收到的信息，再自己思考一遍

所以一个 Transformer block 可以简化理解为：

```text
输入
 -> 多头注意力
 -> 残差 + LayerNorm
 -> 前馈网络
 -> 残差 + LayerNorm
 -> 输出
```

## 九、 原始 Transformer、BERT、GPT：它们是什么关系？

这一点非常值得补充，因为很多人会把它们混成一件事。

### 1. 原始 Transformer（2017 论文）

原始论文提出的是 **Encoder-Decoder** 结构，常用于机器翻译。

- Encoder 负责读输入句子
- Decoder 负责生成输出句子

### 2. BERT

BERT 主要使用的是 **Transformer Encoder**。

特点：

- 双向看上下文
- 擅长理解任务
- 比如分类、抽取、问答

### 3. GPT

GPT 主要使用的是 **Transformer Decoder-only** 结构。

特点：

- 只看左边上下文
- 用 masked self-attention 防止偷看未来 token
- 擅长生成式任务

所以严格说：

- Transformer 是架构家族
- BERT 和 GPT 是它的不同分支

## 十、 Masked Attention：为什么 GPT 不能偷看后文？

如果模型训练目标是“预测下一个 token”，那它绝不能在训练时偷看答案。

比如：

```text
今天 天气 真 [MASK]
```

模型应该根据前文预测 `好`，而不是直接看到 `好`。

所以 GPT 的 self-attention 会加一个 **causal mask（因果掩码）**：

- 当前 token 只能看自己和前面的 token
- 不能看后面的 token

在实现上，通常是把“不该看的位置”在 softmax 前设成极小值（近似负无穷），让它们最终权重变成 0。

这一步不是“数学装饰”，而是保证：

**训练目标和推理时的真实使用方式保持一致。**

***

## 十一、 一句话总结 Attention 在做什么

如果把 Transformer 压缩成一句最直白的话，那就是：

> **每个 token 先带着自己的位置和初始语义进来，然后主动去问：这句话里谁对我最重要？我该从谁那里拿信息？最后把这些信息按权重融合进自己，变成一个更懂上下文的新表示。**

再把这件事重复很多层，模型就越来越能理解：

- 指代
- 语法
- 语义
- 长距离依赖
- 上下文细节

***

## 十二、 对原稿的几个关键修正

为了方便你后续继续写，我把最重要的修正点再集中列一下：

- **`Q/K/V` 不是 token 自带属性**：它们是输入表示经过三组不同权重矩阵映射出来的
- **注意力矩阵不是 “K 行 V 列”**：它是 `Q` 和 `K` 的匹配分数矩阵
- **`V` 不是“初始向量本身”**：而是输入经过 `Wv` 投影后的 value 表示
- **不是“去掉负数得到非线性”**：Attention 里关键归一化步骤是 `softmax`
- **Transformer 不只是 Attention**：还包括位置编码、多头机制、残差连接、LayerNorm、前馈网络等
- **Transformer 不是不看顺序**：它不靠递归顺序处理，但会通过位置编码显式注入顺序信息
